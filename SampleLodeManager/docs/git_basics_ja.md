# Git の「コミット」「プル」「プッシュ」って何？

Git は **変更の履歴**を記録する仕組みです。よく出てくる言葉だけ、短くまとめます。

## コミット（commit）

- **いまのフォルダの状態を、履歴に 1 記録として残す**ことです。
- 例: 「README を直した」「バグを直した」など、まとまった単位でメッセージを付けて保存します。
- まだ **自分の PC の中**だけの話で、この時点では GitHub には載りません。

## プッシュ（push）

- **ローカルに積んだコミットを、GitHub（リモート）に送る**ことです。
- `main` に push すると、GitHub 上の `main` がそのコミットまで進みます。
- 他の人や **GitHub Actions**（自動処理）は、だいたいこのリモート上の履歴を見ます。

## プル（pull）

- **GitHub（リモート）の新しいコミットを、自分の PC に取り込む**ことです。
- 自分以外が push した変更や、**`github-actions[bot]` が push した `index.xml` の更新**などを手元に揃えたいときに使います。
- pull していないと、**GitHub の最新と手元のファイルがズレたまま**になります。

## このリポジトリでの流れ（イメージ）

1. あなたがコードや README を直す → **コミット** → **プッシュ**  
2. GitHub が **Actions**（例: `deploy`）を動かす → `reapack-index` が **`index.xml` を更新**して **bot がコミット＆プッシュ**  
3. あなたが手元のフォルダも最新にしたい → **プル**

## よくある勘違い

- **「push した＝すぐに ReaPack ユーザーに届く」** ではありません。ユーザーは ReaPack で **Synchronize** などして、更新された `index.xml` を取りに行く必要があります。
- **「Actions が緑＝手元のファイルも自動で変わる」** ではありません。手元は **pull** で揃えます。
